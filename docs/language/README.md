# Language guide

A sequential tour of the Spock language. Each page teaches one concept
cluster with complete, checkable programs; together they cover everything the
[v0 specification](../spec/v0.md) defines, in an order meant for learning
rather than lookup.

- [Tables, types, and defaults](tables.md) — durable truth: tables, keys,
  references, closed sets, and validator checks.
- [The derived API](derived-api.md) — the contract, REST and GraphQL
  surfaces, and the error vocabulary your schema mints for free.
- [Functions and refusals](functions.md) — read and write functions, SQL
  escape bodies, and product errors.
- [Seed and disposable state](seed.md) — how a program populates its world,
  and why there are no migrations.
- [Identity: the actor seam](actor.md) — the development identity model and
  the `me` default.
